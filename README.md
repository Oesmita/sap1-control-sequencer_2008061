## Project Overview

This repository contains a complete, enhanced **SAP-1 (Simple-As-Possible) CPU** built and simulated in **Logisim Evolution**. The SAP-1 is a foundational computer architecture used to teach the basic principles of CPU design.

This enhanced implementation extends the classical SAP-1 with:
- A **hardwired Control Sequencer** that automates the fetch–decode–execute cycle without microprogramming
- A **ROM-to-RAM bootloader** that loads programs automatically (2-phase handshake mechanism)
- An **expanded 9-instruction ISA** including rotate, jump, arithmetic-on-store, and halt
- A **Python assembler** that converts human-readable assembly into Logisim ROM hex images
- **Two separate Logisim circuit files** — one for Automatic mode and one for Manual/Loader mode

The unique feature of this design is **arithmetic-on-store semantics**: the `STA` instruction stores A+B to memory and `STS` stores A−B to memory, enabling in-place memory results without needing a separate ADD/SUB opcode.

---

## Features

- **Load Data from Memory:** `LDA` and `LDB` load values into Register A and B
- **Arithmetic-on-Store:** `STA` stores (A+B) and `STS` stores (A−B) directly to memory
- **Bit Rotation:** `ROL_A`, `ROR_A`, `ROL_B`, `ROR_B` rotate register contents left or right
- **Unconditional Jump:** `JUMP` transfers control to any memory address
- **Halt:** `HLT` stops the processor by disabling the ring counter
- **Hardwired Control:** All control signals generated by Boolean logic — no microprogramming
- **6-Phase Ring Counter:** Precise T1–T6 timing for all micro-operations
- **Single-Bus Architecture:** Tri-state logic ensures only one module drives the bus at a time
- **ROM Bootloader:** Auto-loads 16 bytes from ROM to SRAM in 32 clock pulses
- **Manual Debug Mode:** Direct RAM access for step-by-step programming and diagnostics
- **Python Assembler:** Converts SAP-1 assembly to 16-byte hex ROM image

## Architecture Components

The SAP-1 CPU is composed of several fundamental building blocks, all connected through a single shared 8-bit data bus.

### Program Counter (PC)
A 4-bit counter that holds the address of the next instruction to be fetched. It increments automatically after each instruction fetch and can be directly loaded with a new address during a `JUMP` instruction.

### Memory Address Register (MAR)
A 4-bit register that holds the address of the memory location currently being accessed. During fetch (T1), it receives the PC value. During execute (T3), it receives the operand from the Instruction Register.

### SRAM (16 × 8-bit)
The main memory — stores both instructions and data across 16 addresses. Read with `sram_rd = 1`, written with `sram_wr = 1`.

### Instruction Register (IR)
An 8-bit register that holds the currently executing instruction:
- `IR[7:4]` → **opcode** (what operation to perform)
- `IR[3:0]` → **operand** (target memory address)

### Registers A and B
Two 8-bit general-purpose registers built from D flip-flops:
- **Register A** is the main Accumulator — stores results and participates in all arithmetic
- **Register B** holds the second operand for ALU operations
- Both are directly wired to ALU inputs internally (no bus needed for computation)

### Arithmetic Logic Unit (ALU)
An 8-bit unit that performs:
- **Addition:** A + B (default)
- **Subtraction:** A − B (when `alu_sub = 1`)
- **Bit rotation:** left or right, for Register A or B, controlled by `right_or_left` and `rotate_reg_en`

Results are placed on the bus when `alu_out = 1`.

### Timing Control Generator (Ring Counter)
A 6-phase ring counter that cycles through states T1 → T2 → T3 → T4 → T1... Each state activates specific control signals. Stops permanently when `HLT` sets `hlt_set = 1`.

### Opcode Decoder (4-to-16)
Converts the 4-bit opcode from IR into one-hot control signals — one line per instruction. These signals combine with timing pulses to produce all micro-operation enables.

### Bootloader (Auto Mode only)
A 2-phase FSM that transfers all 16 bytes from ROM to SRAM when `debug = HIGH`:
- **Phase 1:** Drives address onto bus → latches into MAR
- **Phase 2:** Drives ROM data onto bus → writes to SRAM → increments counter
- Completes in 32 clock pulses (2 per byte)

## Control Unit (Hardwired)

The control unit is implemented entirely using combinational Boolean logic (AND, OR, NOT gates) combined with the 6-phase ring counter. No microprogramming ROM is used.

The global enable signal `c_en` (also called `cs_en`) activates the sequencer. When `debug = 1`, `c_en` is forced LOW, disabling all automatic CU outputs and giving the bootloader exclusive bus access.

### Key Control Signal Equations

| Signal | Activated When |
|---|---|
| `pc_out` | `T1 AND c_en` |
| `mar_in_en` | `(T1 AND c_en) OR (T3 AND c_en)` |
| `sram_rd` | `(T2 AND c_en) OR (T4 AND (LDA OR LDB) AND c_en)` |
| `ins_reg_in_en` | `T2 AND c_en` |
| `pc_en` | `T2 AND c_en` |
| `ins_reg_out_en` | `T3 AND c_en` |
| `a_in` | `T4 AND LDA AND c_en` |
| `b_in` | `T4 AND LDB AND c_en` |
| `alu_out` | `T4 AND (STA OR STS) AND c_en` |
| `alu_sub` | `T4 AND STS AND c_en` |
| `sram_wr` | `T4 AND (STA OR STS) AND c_en` |
| `ro_out_en` | `T4 AND (ROL_A OR ROR_A OR ROL_B OR ROR_B) AND c_en` |
| `right_or_left` | `T4 AND (ROL_A OR ROL_B) AND c_en` |
| `rotate_reg_en` | `T4 AND (ROL_B OR ROR_B) AND c_en` |
| `pc_load` | `T4 AND JUMP AND c_en` |
| `address_out_en` | `T4 AND JUMP AND c_en` |
| `hlt_set` | `T4 AND HLT AND c_en` |

## Instruction Set Architecture

Each instruction is **8 bits**: upper 4 bits = opcode, lower 4 bits = operand/address.

**Encoding formula:** `byte = (opcode << 4) | operand`  
**Example:** `LDA 0xE` → `(0001 << 4) | 1110` = `0001 1110` = `0x1E`

| Mnemonic | Opcode (Binary) | Hex | Needs Address? | Description |
|---|---|---|---|---|
| `LDA addr` | `0001` | `1` | Yes | Load value from memory into Register A |
| `LDB addr` | `0010` | `2` | Yes | Load value from memory into Register B |
| `STA addr` | `0011` | `3` | Yes | Store (A + B) into memory |
| `STS addr` | `0100` | `4` | Yes | Store (A − B) into memory |
| `ROR_A` | `0110` | `6` | No | Rotate Register A one bit to the right |
| `ROR_B` | `0111` | `7` | No | Rotate Register B one bit to the right |
| `ROL_A` | `1000` | `8` | No | Rotate Register A one bit to the left |
| `ROL_B` | `1001` | `9` | No | Rotate Register B one bit to the left |
| `JUMP addr` | `1010` | `A` | Yes | Unconditional jump to address |
| `HLT` | `1011` | `B` | No | Halt the processor |

## Fetch–Decode–Execute Cycle

Every instruction follows the same 4-step sequence:

```
T1:  MAR ← PC               (send current PC address to MAR for fetch)
T2:  IR  ← M[MAR]           (fetch instruction from SRAM into IR)
     PC  ← PC + 1           (increment PC to point to next instruction)
T3:  MAR ← IR[3:0]          (load operand address into MAR — same for all instructions)
T4:  <execute>              (instruction-specific action)
```

### T4 Execute Actions by Instruction

| Instruction | T4 Action |
|---|---|
| `LDA` | `sram_rd, a_in` → Register A ← M[MAR] |
| `LDB` | `sram_rd, b_in` → Register B ← M[MAR] |
| `STA` | `alu_out, sram_wr` → M[MAR] ← A + B |
| `STS` | `alu_out(sub), sram_wr` → M[MAR] ← A − B |
| `ROL_A` | `ro_out_en, right_or_left` → A rotated left by 1 bit |
| `ROR_A` | `ro_out_en` → A rotated right by 1 bit |
| `ROL_B` | `ro_out_en, right_or_left, rotate_reg_en` → B rotated left by 1 bit |
| `ROR_B` | `ro_out_en, rotate_reg_en` → B rotated right by 1 bit |
| `JUMP` | `address_out_en, pc_load` → PC ← IR[3:0] |
| `HLT` | `hlt_set = 1` → ring counter disabled, processor stops |

## Assembler / Compiler

A Python assembler is provided in the `Compiler/` folder. It converts SAP-1 assembly source code into a 16-byte hex ROM image ready to paste into Logisim.

### Files
- `sap1_asm.py` — command-line assembler
- `Sap1_asm.ipynb` — Jupyter notebook version

### Assembly Syntax

```asm
; Comments use ; or #
ORG  0          ; set current address (0–15)
LDA  1110       ; operands: binary (1110), decimal (14), or hex (0xE or E)
LDB  1111
JUMP 0011
ORG  3
ROL_A           ; no operand needed for rotate/halt
STA  1100
HLT
ORG  14
DATA 00101100   ; raw 8-bit data value stored at this address
DATA 00000111
```

### How to Run

```bash
# Basic — prints 16 hex bytes to console
python sap1_asm.py your_program.asm

# Save to file
python sap1_asm.py your_program.asm --output output.hex

# Show annotated disassembly
python sap1_asm.py your_program.asm --verbose
```

### Main Demo — Rotate with Jump

```asm
ORG 0
LDA  1110       ; A ← M[0xE] = 0x2C = 44
LDB  1111       ; B ← M[0xF] = 0x07 = 7
JUMP 0011       ; jump to rotate routine at address 3

ORG 3
ROL_A           ; A: 0x2C (00101100) → 0x58 (01011000)
ROL_B           ; B: 0x07 (00000111) → 0x0E (00001110)
ROR_A           ; A: 0x58 → 0x2C  (back to original)
ROR_B           ; B: 0x0E → 0x07  (back to original)
STA  1100       ; M[0xC] ← A + B = 44 + 7 = 51 (0x33)
STS  1101       ; M[0xD] ← A − B = 44 − 7 = 37 (0x25)
HLT

ORG 14
DATA 00101100   ; 0x2C = 44
DATA 00000111   ; 0x07 = 7
```

**Assembled ROM image (paste into Logisim ROM):**
```
1E 2F A3 80 90 60 70 3C 4D B0 00 00 00 00 2C 07
```

**Expected results after execution:**

| Location | Value | Meaning |
|---|---|---|
| Register A | `0x2C` (44) | Unchanged after ROL→ROR pair |
| Register B | `0x07` (7) | Unchanged after ROL→ROR pair |
| M[0xC] | `0x33` (51) | A + B stored by STA |
| M[0xD] | `0x25` (37) | A − B stored by STS |
| PC / Ring Counter | Stopped | HLT reached |

## Run the CPU — Auto Mode

Use **`SAP1_auto_2008061.circ`** for this mode.

> Download Logisim Evolution: [logisim-evolution releases](https://github.com/logisim-evolution/logisim-evolution/releases)

### Step 1 — Initial Setup
1. Open `SAP1_auto_2008061.circ` in Logisim Evolution
2. Set `debug = LOW (0)`
3. Set `clk = OFF`
4. Pulse `pc_reset` once → PC resets to `0000`

### Step 2 — Program the ROM
1. Right-click the **ROM** component → **Edit Contents**
2. Enter your 16 hex bytes from the assembler
3. For the demo: `1E 2F A3 80 90 60 70 3C 4D B0 00 00 00 00 2C 07`
4. Close the editor

### Step 3 — Load ROM to RAM (Bootloader)
1. Set `debug = HIGH (1)` → bootloader activates, control sequencer is masked
2. Apply clock pulses:
   - **Phase 1:** address latches into MAR
   - **Phase 2:** ROM data writes to SRAM, counter increments
3. After **32 clock pulses**, all 16 bytes are in SRAM
4. You can watch MAR and Data Bus change with each clock

### Step 4 — Exit Bootloader
1. Set `debug = LOW (0)` → bootloader releases the bus
2. Give one idle clock pulse to let the FSM settle

### Step 5 — Run Automatically
1. Pulse `pc_reset` → PC = `0000`
2. Set `cs_en = HIGH (1)` → control sequencer enabled
3. Apply clock pulses (manual or enable auto-tick in Logisim)
4. The CPU runs through each instruction automatically:
   - **T1:** PC → MAR
   - **T2:** M[MAR] → IR, PC++
   - **T3:** IR[3:0] → MAR
   - **T4:** Execute
5. Each instruction = **4 clock cycles**

### Step 6 — Verify Results
After `HLT`, all values freeze:
- **Register A** = `0x2C` (44)
- **Register B** = `0x07` (7)
- **SRAM[0xC]** = `0x33` (51) → A + B
- **SRAM[0xD]** = `0x25` (37) → A − B

## Run the CPU — Manual Mode

Use **`SAP1_manual.circ`** for this mode. In this mode you manually enter each byte into RAM using the debug pins — no ROM or bootloader is used.

> Download Logisim Evolution: [logisim-evolution releases](https://github.com/logisim-evolution/logisim-evolution/releases)

### Step 1 — Initial Setup
1. Open `SAP1_manual.circ` in Logisim Evolution
2. Set `debug = LOW (0)`
3. Set `cs_en = HIGH (1)`
4. Pulse `pc_reset` → PC = `0000`
5. Set `clk = OFF`

### Step 2 — Program RAM Manually (Debug Mode)
1. Set `debug = HIGH (1)` → automatic control disabled, manual pins active
2. For **each byte**, repeat:
   - Set `debug_data` = **8-bit memory address** (e.g., `00000000` for address 0)
   - Pulse `mar_in_en_manual` → address latches into MAR
   - Set `debug_data` = **8-bit instruction or data** (e.g., `00011110` for `LDA 0xE`)
   - Pulse `sram_wr_manual` → value written to SRAM

**Demo program — byte by byte:**

| Address (binary) | Data (binary) | Meaning |
|---|---|---|
| `00000000` | `00011110` | `LDA 0xE` |
| `00000001` | `00101111` | `LDB 0xF` |
| `00000010` | `10100011` | `JUMP 0x3` |
| `00000011` | `10000000` | `ROL_A` |
| `00000100` | `10010000` | `ROL_B` |
| `00000101` | `01100000` | `ROR_A` |
| `00000110` | `01110000` | `ROR_B` |
| `00000111` | `00111100` | `STA 0xC` |
| `00001000` | `01001101` | `STS 0xD` |
| `00001001` | `10110000` | `HLT` |
| `00001110` | `00101100` | DATA: 44 (0x2C) |
| `00001111` | `00000111` | DATA: 7 (0x07) |

### Step 3 — Exit Debug Mode
1. Set `debug = LOW (0)`
2. Pulse `pc_reset` → PC = `0000`

### Step 4 — Run the Program
1. Confirm `cs_en = HIGH (1)`
2. Apply clock pulses
3. Processor halts automatically at `HLT`

### Step 5 — Verify Results
Same as Auto Mode — check Register A, Register B, SRAM[0xC] and SRAM[0xD].
